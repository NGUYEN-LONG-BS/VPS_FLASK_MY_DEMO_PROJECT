Hướng Dẫn Điều Hướng Trong Next.js 15

Điều hướng từ page.tsx sang file HTML tĩnh với App Router

Tổng Quan

Trong Next.js 15 với App Router, bạn có thể điều hướng đến file HTML tĩnh trong thư mục public/ bằng cách sử dụng Link, useRouter, hoặc thẻ <a>. Dưới đây là các cách thực hiện với giao diện đẹp mắt.

Các Cách Điều Hướng

Sử dụng Link

Sử dụng component Link từ next/link để điều hướng phía client mà không tải lại trang.

Sử dụng useRouter

Dùng hook useRouter từ next/navigation để điều hướng bằng lập trình, ví dụ khi nhấn nút.

Sử dụng thẻ <a>

Liên kết trực tiếp bằng thẻ <a>, nhưng sẽ gây tải lại toàn bộ trang.

Thử với thẻ <a>

Ví Dụ Mã Nguồn

Dưới đây là mã nguồn ví dụ cho file page.tsx để điều hướng đến example.html.

Cách 1: Sử dụng Link từ next/link Next.js cung cấp component Link để điều hướng phía client mà không cần tải lại toàn bộ trang. Trong file page.tsx, bạn có thể sử dụng Link để trỏ đến file HTML trong thư mục public/: 

    

// app/page.tsx
import Link from 'next/link';

export default function HomePage() {
return (
    <div>
    <h1>Trang chính</h1>
    <Link href="/example.html">Đi đến file HTML</Link>
    </div>
);
}

Lưu ý: Đường dẫn trong href phải tương ứng với vị trí của file HTML trong thư mục public/. Ví dụ, nếu file nằm trong public/example.html, bạn sử dụng /example.html.

Cách 2: Sử dụng useRouter từ next/navigation Nếu bạn cần điều hướng bằng lập trình (ví dụ, khi người dùng nhấn nút hoặc sau một sự kiện), bạn có thể sử dụng hook useRouter để chuyển hướng đến file HTML. 

    
// app/page.tsx
'use client'; // Bắt buộc để sử dụng các hook phía client
import { useRouter } from 'next/navigation';

export default function HomePage() {
  const router = useRouter();

  const handleNavigate = () => {
    router.push('/example.html');
  };

  return (
    <div>
      <h1>Trang chính</h1>
      <button onClick={handleNavigate}>Đi đến file HTML</button>
    </div>
  );
}

Lưu ý: Thêm directive 'use client' ở đầu file vì useRouter chỉ hoạt động trong các component phía client.

Cách 3: Sử dụng thẻ thông thường Nếu bạn không cần điều hướng phía client (client-side navigation) và chỉ muốn liên kết trực tiếp đến file HTML, bạn có thể sử dụng thẻ HTML <a> 

    

// app/page.tsx
export default function HomePage() {
  return (
    <div>
      <h1>Trang chính</h1>
      <a href="/example.html">Đi đến file HTML</a>
    </div>
  );
}

Lưu ý: Sử dụng <a> sẽ gây ra tải lại toàn bộ trang (full page reload), không tận dụng được điều hướng phía client của Next.js. Nên ưu tiên Link hoặc useRouter nếu muốn trải nghiệm mượt mà hơn.

Lưu Ý Quan Trọng

  • Đặt file HTML trong thư mục public/ (ví dụ: public/example.html).
  • Đường dẫn trong href hoặc router.push phải khớp với vị trí file HTML.
  • Sử dụng Link hoặc useRouter để điều hướng phía client, tránh tải lại trang.
  • File HTML tĩnh không được xử lý bởi App Router, chỉ phục vụ như tài nguyên tĩnh.

    • Thư mục public/: Đảm bảo file HTML được đặt đúng trong thư mục public/. Next.js sẽ tự động phục vụ các file trong public/ dưới dạng tài nguyên tĩnh.

    Đường dẫn: Đường dẫn đến file HTML phải chính xác. Ví dụ, nếu file nằm trong public/docs/example.html, bạn cần sử dụng /docs/example.html trong Link hoặc useRouter.

    File HTML tĩnh: File HTML sẽ được phục vụ như một tài nguyên tĩnh, không được xử lý bởi hệ thống route của Next.js (App Router). Do đó, nó không có các tính năng như server-side rendering (SSR) hoặc static generation (SSG).

    SEO và hiệu suất: Nếu file HTML chứa nội dung quan trọng, bạn nên cân nhắc chuyển đổi nó thành một trang Next.js (sử dụng page.tsx) để tận dụng các tính năng tối ưu hóa SEO và hiệu suất của Next.js.
